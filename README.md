### Computer Science

- `프로세스와 스레드 🔥`

  - 프로세스가 뭔가요?
    - 실행 중인 프로그램을 말해요. 예를 들어, 컴퓨터에서 프로그램을 켜면 그게 바로 프로세스에요.
  - 스레드가 뭔가요?
    - 프로세스 내에서 실제로 작업을 수행하는 작은 실행 단위입니다. 하나의 프로세스는 여러 스레드를 가질 수 있어요.
  - 프로세스와 스레드는 어떤 차이가 있나요?
    - 프로세스는 실행중인 프로그램을 뜻하고, 스레드는 그 프로세스 안에서 작업되는 무언가를 얘기한다.
    - ex) 프로세스 : 크롬, 스레드 : 크롬에서 유튜브를 시청, 게임 다운로드 등등

- `싱글 스레드와 멀티 스레드 🔥`

  - 싱글 스레드
    - 한 번에 한 작업만 처리할 수 있습니다. 따라서 동기화 문제를 걱정할 필요는 없지만 동시에 여러 작업을 할 수 없다는 단점이 있습니다.
  - 멀티 스레드
    - 한 번에 여러 작업을 할 수 있지만, 한 스레드의 동기화 문제나 자원공유에 신경을 써야한다는 단점이 있습니다.
  - 그렇다면 JS는 싱글? 멀티? 스레드?
    - 우선 답을 먼저 말씀드리자면 JS는 한 가지 작업을 순차적으로 실행하는 싱글 스레드 입니다. 하지만 JS에서 비동기 처리 방식이 존재하기 때문에 멀티 스레드 처럼 보일 수는 있습니다. 이벤트 루프라는 메커니즘 덕분에, 자바스크립트는 비동기 작업을 처리하면서도 다른 작업을 계속 수행할 수 있습니다.

- `HTTP 🔥`

  - HTTP란 뭔가요?
    - HTTP는 통신규약으로, 인터넷에서 데이터를 주고받는 규칙입니다. 웹 브라우저와 서버간의 소통 방식이라고 볼 수 있겠습니다.
  - HTTP 프로토콜의 가장 큰 특징은 뭔가요?
    - HTTP 메세지는 HTTP 서버와 HTTP 클라이언트로 의해 해석이 된다.
    - 클라이언트에서 요청을 보냈을 시, 서버는 그 요청에 알맞은 응답을 보낸다.
  - URL은 뭔가요?
    - 웹 브라우저의 고유 리소스 주소를 의미합니다.
  - HTTP/1.1 과 HTTP/2.0의 차이는 뭔가요?
    - HTTP/1.1: 싱글 태스킹 (한 번에 하나의 요청만 처리 가능) / 브라우저가 요청해야지만 서버 푸쉬가 가능
    - HTTP/2.0: 멀티 태스킹 (한 번에 여러개의 요청을 처리 가능) / 미리 서버쪽에서 클라이언트가 필요할 리소스를 미리 푸시해 줄 수 있 수 있습니다.
  - HTTPS는 HTTP랑 뭐가 다른가요?
    - SSL 인증을 했냐, 하지 않았느냐의 차이인데, HTTPS는 SSL 인증을 함으로써 데이터를 암호화하여 더 안전하게 사용 가능합니다.
  - 심화) 공개키 (비대칭키) 방식이 뭔가요?
    - 대칭키: 하나의 동일한 암호키로 데이터를 암호화하고 복호화할 수 있습니다.
    - 비대칭키: 두 개의 서로 다른 키를 사용합니다: 하나는 공개키(Public Key), 다른 하나는 개인키(Private Key)입니다. 공개키로 암호화 된 데이터는 개인키로만이 복호화 할 수 있으며, 개인키로 암호화 한 데이터는 공개키로 복호화할 수 있습니다.
      - 데이터의 암호화 (기밀성 유지)
      - 디지털 서명 (신뢰성과 무결성 보장)

- `CI CD 🔥`

  - CI CD란 뭔가요?
    - CI는 Contiunous Integration의 약자로, 지속적 통합을 의미하는데, 프로그래머들이 각자가 만든 코드를 통합하는 단계를 말하기도 한다.
    - CD는 Continuous Deployment의 약자로, 지속적 배포를 의미한다. 소프트웨어를 코딩한 결과를 최종 사용자에게 실행 가능하도록 하는 단계를 말하기도 한다.

- `웹팩 🔥`

  - 웹팩이란?
    - 여러 자바스크립트 파일을 하나로 묶어 효율적으로 관리해주는 도구입니다. 이를 통해 로딩 속도를 빠르게 할 수 있다는 장점이 있습니다.
    - 필요한 다수의 자바스크립트 파일을 하나의 자바 스크립트 파일로 만들어 주는 것을 Webpack 이라고 한다.
  - 모듈이란?
    - 개발하는 애플리케이션이 크기가 커지면서 확장성과 유지보수를 위하여 파일을 여러개로 분리해야하는데, 이 때 각 파일 하나하나를 모듈이라고 부른다.
    - 모듈은 보통 클래스 하나 혹은, 특정 목적을 가지는 복수의 함수 구성을 뜻하기도 한다.
  - 모듈 번들링이란?
    - 세분화 되어 분리된 모듈들을 묶어주는 것을 의미한다
  - 웹팩이 등장한 이유 웹팩 사용 시에 이점
    - 파일 단위의 모듈 관리 필요성
    - 애플리케이션의 빠른 로딩 속도와 높은 성능
  - 바벨이란?
    - 입력과 출력 모두 자바스크립트 코드인 컴파일러를 의미한다.
  - 웹팩의 주요 속성 4가지
    - Entry : 빌드를 할 대상 파일 정의
    - Output : 빌드를 한 후(=웹팩으로 변환 후)의 결과물에 대한 정보를 저장
    - Loader(모듈) : entry -> output 단계에서 개입
    - Plugins

- `타입과 인터페이스 🔥`
  - 타입스크립트를 왜 쓰나요? (본인이 느낀점)
    - 유연한 자바스크립트가 잡아내지 못 하는 에러를 타입스크립트에선 타입만 지정해주면 빠르게 확일 할 수 있기 때문에
  - 타입과 인터페이스의 차이를 아나요?
    - 타입은 선언 병합이 불가능하고, 인터페이스는 가능하다.

### Javascript

- `프로그래밍 🔥`

  - 프로그래밍이란 뭐라고 생각하나요?
    - 프로그래밍은 컴퓨터가 해야 할 일을 사람이 이해할 수 있는 언어로 지시하는 과정입니다.
  - 컴파일러는 뭐고 인터프리터는 뭔가요?
    - 프로그래밍 언어로 작성된 코드를 컴퓨터가 이해할 수 있게 컴퓨터 언어로 변환해주는 변환기
    - 컴파일러 : 코드를 한 번에 변환해서 실행합니다.
    - 인터프리터 : 작성된 코드를 한 줄씩 순차적으로 읽고, 그 즉시 해석하고 실행합니다.
  - 자바스크립트란 🔥
    - 웹페이지를 풍부하게 만들어주는 스크립트 혹은 프로그래밍 언어이다.
  - 자바스크립트의 특징은 뭐가 있나요?
    - 객체기반 언어이지만 상속과 클래스 개념은 존재하지 않는다.
    - 컴파일 과정이 없기 때문에 다른 언어들에 비해 빠른 시간안에 코드를 작성할 수 있다.
    - HTML문서 내에 기술되고 HTML 문서와 함께 수행된다.

- `변수 🔥`

  - 변수란 무엇인가요?
    - 하나의 값을 저장할 수 있는 저장공간을 변수라고 한다.
  - 식별자란 무엇인가요? 🔥
    - 자바스크립트에서 이름을 붙일 때 사용하는 단어
    - 예시로, 변수명과 함수명이 존재
  - 호이스팅이 뭔가요? 🔥🔥🔥🔥
    - 변수와 함수 선언이 실제 코드 실행 전에 스코프의 최상단으로 끌어올려지는 현상
  - var, let, const 차이점
    - var vs let, const : 중복 선언이 불가능 하다 / 모든 코드 블록을 지역 스코프로 인정하는 블록 레벨 스코프를 따른다.
    - let vs const : let은 선언 단계와 초기화 단계가 분리되어 진행되지만 const는 동시에 진행된다.
  - TDZ 🔥🔥🔥
    - 선언 단계와 초기화 단계 사이를 일시적 사각 지대라고 부른다.
  - 식별자 네이밍 규칙은 어떤 것들이 있나요?
    - 식별자는 특수문자를 제외한 문자, 숫자, 언더스코어(\_), 달러 기호($)를 포함할 수 있다.
    - 단, 식별자는 특수문자를 제외한 문자, 언더스코어(\_), 달러 기호($)로 시작해야 한다. (숫자 X)
    - 예약어는 식별자로 사용할 수 없다.
  - 네이밍 컨벤션은 어떤 것들이 있나요?
    - 하나 이상의 영어 단어로 구성된 식별자를 만들 때 가독성 좋게 단어를 한눈에 구분하기 위해 규정한 명명 규칙
    - Camel Case / Pascal Case

    - 값을 참조할 때 한 번에 읽어 들여야 할 메모리 공간의 크기를 결정하기 위해
  - 정적 타이핑이 뭔가요?

    - map : 배열 내의 모든 요소 각각에 대하여 주어진 함수를 호출한 결과를 모아 새로운 배열을 반환한다.

- `브라우저 렌더링 과정 🔥`

  - 브라우저의 렌더링 과정에 대해 설명해보세요 🔥
    - 브라우저는 HTML, CSS, 자바스크립트, 이미지 폰트 파일 등 렌더링에 필요한 리소스를 요청하고 서버로부터 응답 받음
    - 브라우저의 렌더링 엔진은 서버로부터 응답된 HTML과 CSS를 파싱하여 DOM과 CSSOM을 생성하고 이들을 결합하여 렌더 트리를 생성
    - 브라우저의 자바스크립트 엔진은 서버로부터 응답된 자바스크립트를 파싱하여 AST를 생성하고 바이트코드로 변환하여 실행. 이때 자바스크립트는 DOM API를 통해 DOM이나 CSSOM을 변경할 수 있다. 변경된 DOM과 CSSOM은 다시 렌더 트리로 결합됨
    - 렌더 트리를 기반으로 HTML 요소의 레이아웃을 계산하고 브라우저의 화면에 HTML 요소를 페인팅
  - 브라우저의 렌더링 과정에 자바스크립트는 어떻게 동작하나요? 🔥
    - script 태그를 만나게 될 때에도 DOM 생성이 중단
    - 이후 서버에서 자바스크립트 파일을 응답 받으면, 자바스크립트 엔진에 제어권이 넘어감
    - 자바스크립트 엔진은 자바스크립트 파싱과 실행을 처리
    - 자바스크립트 코드를 해석하여 AST(Abstract Syntax Tree, 추상적 구문 트리)를 생성
    - 이후 AST를 바탕으로 바이트코드를 생성하고 실행
  - <script></script> 태그를 <body></body> 태그 밑에 둬야하는 이유가 있을까요?
    - HTML을 읽는 과정에 스크립트를 만나면 중단 시점이 생기고 그만큼 Display에 표시되는 것이 지연된다.
    - DOM 트리가 생성되기전에 자바스크립트가 생성되지도 않은 DOM의 조작을 시도할 수 있다.

    - 현재 이벤트의 기본 동작을 중단

    - 현재 이벤트가 상위로 전파되지 않도록 중단

    - clearTimeout : 반환받은 id 값을 이용하여 setTimeout을 취소
    - setInterval : 지정한 시간 간격을 두고 지정한 함수 계속 실행
    - clearInterval : 반환받은 id 값을 이용하여 setInterval을 취소
  - 이벤트가 과도하게 호출되어 성능에 문제를 일으킬 경우에 할 수 있는 어떤 일을 통해 해결할 수 있나요?

  - 쓰로틀에 대해서 알고 있나요?
    - 짧은 시간 간격으로 이벤트가 연속해서 발생하더라도 일정 시간 간격으로 이벤트 핸들러가 최대 한 번만 호출되도록 한다.

- `비동기 프로그래밍 🔥`

  - 동기와 비동기의 차이점에 대해서 설명해줄 수 있나요? 🔥🔥
    - 동기는 순차적, 직렬적으로 테스크를 수행하고, 비동기는 병렬적으로 테스크를 수행한다.
  - 이벤트 루프와 태스크 큐에 대해서 알고 있나요? 🔥🔥🔥
    - 이벤트 루프 : 여러 스레드가 사용되는 구동 환경이 자바스크립트 엔진과 연동하기 위해 사용되는 장치
    - 태스크 큐 : 컴퓨터의 기본적인 자료 구조 중 한 가지로, 먼저 집어 넣은 데이터가 먼저 나오는 FIFO(First In First Out) 구조로 저장하는 형식

- 마이크로태스크 큐에 대해서 알고 있나요? 🔥🔥

  - 비동기 함수가 실행되면, Web API가 호출된다. Web API는 비동기 함수의 콜백함수를 콜백큐에 넣는다. 비동기 콜백에 따라 Promise는 마이크로태스크 큐로, setTimeout은 태스크 큐로 가게된다. 이벤트 루프는 Call stack이 비어있는지 확인하고, 마이크로 태스크 큐에 가장 오래 담겨있던 콜백함수를 콜스택으로 보낸다. 콜스택이 비었다면 그 후에 태스크 큐를 확인하여 콜백함수를 콜스택으로 밀어넣는다.

- 태스크 큐와 마이크로태스크 큐 중 어떤 것이 먼저 실행되나요? 🔥🔥
  - 마이크로태스크 큐가 우선적으로 실행된다. (콜백함수는 setTimeout()의 콜백함수보다 먼저 처리된다.)

    - 다른 함수의 매개변수로 함수를 전달하고, 어떠한 이벤트가 발생한 후 매개변수로 전달한 함수가 다시 호출되는 것
  - 프로미스가 뭔가요? 🔥
    - 비동기 처리에 사용되는 객체
  - 프로미스 생성 방법
    - new Promise((resolve, reject) => {})
  - 프로미스의 상태를 나타내는 것은 어떤 것들이 있나요? 🔥
  - Pending
  - Fullfilled
  - Rejected
  - 프로미스 빌트인 객체가 제공하는 정적 메서드에 대해 알고 있나요? 🔥
    - promise.resolve
    - promise.reject
    - promise.all

- `async await 🔥`

  - async/await 가 뭔가요? 기존의 Promise와는 어떤 차이가 있죠? 🔥
    - 에러 핸들링 : promise는 .catch 사용, async는 try-cacth 사용
    - 코드 가독성 : promise는 .then() 지옥 가능성이 높다. 하지만 async는 방지해준다.

### React

- `리액트는 라이브러리인가요 프레임워크 인가요?🔥`

  - 리액트는 라이브러리이다.
  - 라이브러리 : 라이브러리를 가져다가 사용하고 호출하는 측에 주도권을 가지고 있는 것
  - 프레임워크 : 이미 만들어진 틀에 따라 개발자가 어플리케이션 코드를 짜게하는 것

- `리액트를 사용하는 이유🔥🔥`

  - component 단위 작성 : 컴포넌트는 개별적인 뷰 단위로써, 유지보수 및 재생산성이 용이하다.
  - JSX : javascript 확장 구문으로, HTML과 javascript 파일을 따로 관리하지 않아도 된다는 편의성을 가지고 있다.

- `virtual DOM에 대해서 아나요?🔥🔥`

  - DOM : HTML 혹은 XML을 파싱한 것으로 유저들에게 브라우저를 통해서 직접적으로 보여주는 화면을 제공하는 것이다.
  - virtual Dom : DOM이 생성되기 전, 이전 상태 값과 수정사항을 비교하여 달라진 부분만 DOM에게 한 번에 전달하여 딱 한 번만 렌더링 하는 것이다.

- `React에서 함수형 컴포넌트와 클래스형 컴포넌트의 차이 🔥`

  - 선언방식 차이 : class는 extends Component와 render()를 해줘야 한다.
  - State 사용 차이 : 클래스는 this.state, this.setState를 사용하는 반면, 함수는 useState를 사용한다.
  - props 사용 차이 : 클래스는 this.props를 사용하는 반면, 함수는 그냥 사용이 가능하다.
  - LifeCycle 사용여부 : 클래스는 사용 / 함수는 사용불가 (lifeCycle이란 컴포넌트가 DOM위에 생성되기 전후 및 데이터 변화에 상태 업데이트 전후로 실행되는 메소드를 말한다.)

- `props와 state의 차이🔥`

  - props : 외부에서 상속받는 데이터로, 불가변 데이터이다.
  - state : 컴포넌트 내부에서 생성하고 가변 데이터이다.

- `Props가 컴포넌트간에 전달받는 것이라고 했는데 자식에서 부모로도 전달할 수 있는가 🔥`

  - props를 직접적으로 전달해주기 보다는, setState로 자식에게서 부모로 props가 변경 가능하다.
  - 하지만 react는 단방향 데이터 흐름을 원칙하에 두기 때문에 직접적으로 부모에게 전달 할 수는 없다.

- `FLUX에 대해서 아나요? 🔥🔥`

  - 애플리케이션의 데이터 흐름을 관리하는 패턴을 말한다.

- `리덕스에 대해서 아나요? 🔥🔥`

  - 데이터의 상태를 관리해주는 라이브러리이다.

- `리덕스의 기본 원칙은? 🔥🔥`

  - 전체 상탯값을 하나의 객체에 저장
  - 상탯값은 불변 객체
  - 상탯값은 순수 함수에 의해서만 변경되어야 함

- `React에서 state의 불변성을 유지하라는 말이 있는데 이에 대해 설명해달라 🔥`

  - 불변성이란 값이나 상태를 변경할 수 없는 것
  - 함수형 프로그래밍을 지향하기 위해서, 그리고 상태 업데이트를 효율적으로 하기 위해서
  - 함수형 프로그래밍은 순수함수를 조합하여 프로그램의 사이드 이펙트를 방지함

- `리듀서 내부에서 불변성을 지키는 이유는? 전개 연산자의 단점을 해결할 수 있는 방법은 무엇인가 🔥`

  - 가상돔에서 변경 된 값을 감지해야 DOM에서 리렌더링 될 수 있는데, 불변성이 지켜지지 않으면 값이 바뀐지 모르기 때문에 에러가 난다.

- `리액트 사용시에 부수효과로 인해 생기는 문제점이 있다면 🔥🔥`

  - side effect : 함수 내에서 어떤 구현이 외부에 영향을 미치는 것을 의미
  - 실행상태를 예측하기 어렵게 하며 개발비용을 증가시킨다.

- `부수 효과를 일으키는 함수 (불순 함수)`

  - splice
  - sort
  - push

- `부수 효과를 일으키지 않는 함수 (순수 함수)`

  - 고차함수
  - slice

- `컴포넌트의 라이프 사이클 메서드 🔥🔥`

  - 컴포넌트를 처음으로 렌더링할 때(또는 컴포넌트를 업데이트하기 전후) 특정 작업을 처리해야 하는 경우, 불필요한 업데이트를 방지해야 하는 경우
  - render() : 컴포넌트의 모양새를 정의
  - constructor() : 컴포넌트의 생성자 메서드. 컴포넌트를 만들 때 처음으로 실행
  - componentDidUpdate() : 리렌더링 완료 후 실행
  - componentWillUnmout() : 컴포넌트를 DOM에서 제거할 때 실행
  - 기타등등

- `Hooks의 종류 🔥🔥`

  - `useState`
    - 컴포넌트에서 바뀌는 상태값을 관리
  - `useEffect`
    - 컴포넌트가 렌더링 될 때마다 side effect가 실행될 수 있도록 해주는 것
  - `useReducer`
    - useState처럼 State를 관리하고 업데이트 할 수 있는 Hook
    - useState와 차이점이 있다면, 내부에서 state 업데이트 로직이 있는 반면, useReducer는 외부에 state 업데이트 로직이 있다.
  - `useMemo`
    - 메모이제이션 : 기존에 수행한 연산의 결과값을 어딘가에 저장해두고 동일한 입력이 들어오면 재활용하는 프로그래밍 기법
    - 잦은 렌더링으로 인해 불필요한 로직의 연산이 중복으로 수행되기도 하는데, 이러한 성능 저하 문제를 해결하기 위한 Hook
  - `useCallback`
    - 함수와는 상관 없는 상태값이 변할 때, 함수 컴포넌트에서 불필요하게 함수를 업데이트 하는 것을 방지해 줌
  - `useRef`
    - 특정 DOM을 가리킬 때 사용하는 Hook
    - 포커스 설정, 특정 엘리먼트의 크기/색상 변경
  - `커스텀 Hooks`
    - 기존 내장되어있는 hooks가 아닌 개발자가 직접 만드는 hook
    - 로직의 반복을 최소화 하고 재사용성을 높이기 위함
  - `useMemo와 useCallback의 차이를 아나요 🔥🔥`
    - useMemo는 상태값을 반환하고, useCallback은 함수를 반환
  - `리액트에서 setState는 비동기 동작인가요 동기 동작인가요? 🔥`
    - 비동기 동작이다.
  - `setState가 비동기 동작을 취했을 때 얻을 수 있는 이점은 무엇인가요? 🔥`
    - setState로 state가 변경될때마다 새롭게 render 된다는 점을 생각해보면 여러번의 render를 한번으로 줄일 수 있다.
    - setState를 만나게되면 인자로 전달받은 객체를 하나로 합친 뒤에 업데이트한다.
  - `useLayoutEffect는 무엇인가요? 🔥`
    - useEffect와 비슷한 역할을 한다.
    - 렌더링 후 화면이 업데이트 되기 전에 동기적으로 실행한다.

- `리액트의 성능개선 방법에 대해서 설명해주세요`

  - 컴퍼넌트를 렌더링하고 결과를 메모이징(Memoizing)한다. 그리고 다음 렌더링이 일어날 때 props가 같다면, React는 메모이징(Memoizing)된 내용을 재사용함으로써 불필요한 리렌더링을 피할 수 있다.
